# הבית צריך — פריסה עם סנכרון משותף (Cloudflare)

מערכת אחת, קישור אחד, מסונכרן לכל מי שנכנס. הרשימה נשמרת ב-Cloudflare KV;
האפליקציה ממשיכה לעבוד גם לא-מקוון (מטמון localStorage) ומסנכרנת ברגע שיש רשת.

## מבנה התיקייה לפריסה

```
habayit/
├─ public/
│  ├─ index.html
│  └─ app.js
├─ worker.js
└─ wrangler.jsonc
```

צור תיקיית `public/` והעבר אליה את `index.html` ו-`app.js`.
את `worker.js` ו-`wrangler.jsonc` השאר בשורש.

## פריסה (פעם אחת)

דרוש חשבון Cloudflare (חינמי) ו-Node מותקן.

```bash
# 1. התחברות
npx wrangler login

# 2. יצירת מאגר KV משותף — העתק את ה-id שמתקבל
npx wrangler kv namespace create LIST_KV

# 3. הדבק את ה-id בתוך wrangler.jsonc (במקום PUT_YOUR_KV_NAMESPACE_ID_HERE)

# 4. פריסה
npx wrangler deploy
```

בסיום תקבל כתובת כמו `https://habayit.<שם>.workers.dev`.
זה הקישור לשיתוף — כל מי שפותח אותו רואה ועורך את אותה רשימה, מסונכרן.

## איך הסנכרון עובד

- בטעינה: האפליקציה מושכת את הרשימה מהשרת. אם השרת ריק עדיין — היא זורעת אותו.
- בכל שינוי: נשמר מיד מקומית, ונשלח לשרת אחרי חצי שנייה.
- כל ~4 שניות: בדיקה אם חבר עדכן משהו, ואם כן הרשימה מתעדכנת אצלך.
- ללא רשת: הכול ממשיך לעבוד מקומית ומסתנכרן כשהרשת חוזרת.

## נקודות חשובות

- **כל מי שיש לו את הקישור יכול לראות ולערוך.** לרשימת בית משותפת זה בדיוק הרצוי.
  אם תרצה הרשאות/משתמשים — זה צעד נוסף (אפשר להוסיף סיסמה פשוטה ל-Worker).
- **כתיבה אחרונה מנצחת** אם שני אנשים עורכים בו-זמנית את אותו רגע.
- אם תעדכן את האפליקציה (app.js חדש): החלף את הקובץ ב-public/ והרץ שוב `npx wrangler deploy`.

## הרצה מקומית לבדיקה

```bash
npx wrangler dev
```

זה מריץ הכול (כולל ה-API וה-KV) בכתובת מקומית.
